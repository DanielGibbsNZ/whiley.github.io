The Whiley Compiler

Getting Started

Get started with Whiley by downloading the latest version of the compiler from Github by either:

  • Forking the repository on Github.
  • Downloading a tarball or zipball of the repository.
  • Cloning the repository: git clone https://github.com/Whiley/WhileyCompiler.git

The Whiley Compiler can be built via the command line using Apache Ant by running the ant command from the root of the project, or it can be imported into Eclipse by using the following steps:

  • Click File -> New -> Java Project.
  • Enter the project name as "WhileyCompiler".
  • Uncheck Use default location.
  • Set the location to the location of the WhileyCompiler project.
  • DO SOMETHING HERE TO SETUP THE MODULES.
  • Click Finish.

Directory structure

As well as a config file and build file, the project root contains several directories:

  • bin contains scripts to run the Whiley Compiler and utilities.
  • examples contains example Whiley programs.
  • lib contains libraries used by the Whiley Compiler project.
  • misc contains miscellaneous tools and scripts, such as those for Emacs and Vim.
  • modules contains the source code for the various modules of the Whiley Compiler, as explained below.
  • tests contains test Whiley programs to verify the compiler.

Modules

The modules directory contains the source code for the following modules of the Whiley Compiler:

  • wybs is the SOMETHING.
  • wyc is the compiler front-end, including all code for the lexer, the parser, the raw AST, and various tools for instantiating and configuring a Whiley Compiler.
  • wycc is the SOMETHING.
  • wycs is the SOMETHING.
  • wyil is the Whiley Intermediate Language (wyil). The Whiley Compiler (wyc) converts Whiley programs into this intermediate language, and then various transforms are applied. The wyil constitutes a bytecode (similar, in some ways, to Java Bytecode) which can be interpreted and executed on its own (if desired). This package includes code for performing type checking, constant propagation, and various other manipulations. Furthermore, all the necessary code for manipulating structural types in Whiley is included here. In short, this package constitutes the bulk of the compiler.
  • wyjc is the Whiley-to-Java Compiler. This extends the Whiley Compiler (wyc) with additional transforms for converting wyil code into Java bytecode.
  • wyrl is the SOMETHING.
  • wyrt is the Whiley Standard Library. This provides the core implementation for the standard library. All architecture specific code is omitted and, instead, is stored within the respective back-end module (e.g. wyjc).
